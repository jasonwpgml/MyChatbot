# Streamlit AI 챗봇 미니 프로젝트 요구사항 명세서

## 1. 프로젝트 개요

코딩 Agent를 활용하여 **Streamlit 기반 AI 챗봇 웹 애플리케이션**을 구현하시오.

챗봇은 OpenAI API와 연결되어 사용자의 질문에 답변해야 한다. 채팅방과 대화 내용은 SQLite 데이터베이스에 저장하여, 프로그램을 종료한 뒤 다시 실행해도 이전 기록이 유지되도록 구현한다.

이번 프로젝트의 목표는 다음과 같다.

- Streamlit을 활용한 챗봇 UI 구현
- OpenAI API를 활용한 AI 답변 생성
- SQLite를 활용한 데이터 영구 저장
- 채팅방과 메시지의 1:N 관계 이해
- 코딩 Agent가 작성한 코드를 검토하고 동작 원리 설명

---

## 2. 개발 환경

다음 기술을 사용하시오.

- Python
- Streamlit
- OpenAI API
- SQLite
- 코딩 Agent

OpenAI API 키는 `.env` 파일 또는 환경 변수로 관리한다.

주의:

- API 키를 Python 코드에 직접 작성하지 않는다.
- 실제 API 키가 들어 있는 `.env` 파일을 제출하지 않는다.
- 필요한 패키지는 `requirements.txt`에 기록한다.
- 다른 환경에서도 실행할 수 있도록 `.env.example`을 작성한다.

---

## 3. 필수 구현 요구사항

### 요구사항 1. OpenAI API 기반 챗봇을 구현하시오.

1. 사용자가 채팅 메시지를 입력할 수 있어야 한다.
2. 입력한 메시지를 OpenAI API에 전달해야 한다.
3. AI가 생성한 답변을 채팅 화면에 표시해야 한다.
4. 사용자 메시지와 AI 답변을 구분하여 표시해야 한다.
5. API 키가 없거나 API 호출에 실패하면 오류 내용을 화면에 안내해야 한다.

---

### 요구사항 2. SQLite 데이터베이스를 구성하시오.

채팅방과 메시지를 저장할 SQLite 데이터베이스를 구성하시오.

다음 조건을 만족해야 한다.

- 채팅방 테이블과 메시지 테이블을 분리한다.
- 하나의 채팅방에 여러 메시지가 저장되는 1:N 관계를 구성한다.
- 채팅방에는 채팅방 이름과 생성 시각을 저장한다.
- 메시지에는 소속 채팅방, 역할, 내용, 생성 시각을 저장한다.
- 역할은 사용자 메시지와 AI 메시지를 구분할 수 있어야 한다.
- 프로그램 실행 시 필요한 테이블이 없으면 자동으로 생성한다.

채팅 기록의 기준은 Streamlit 화면이나 임시 변수가 아닌 **SQLite 데이터베이스**여야 한다.

---

### 요구사항 3. 채팅 기록을 영구 저장하시오.

1. 사용자가 입력한 메시지를 데이터베이스에 저장하시오.
2. OpenAI API가 생성한 답변도 데이터베이스에 저장하시오.
3. 메시지를 생성 시각 순서대로 화면에 표시하시오.
4. 프로그램을 종료한 뒤 다시 실행해도 기존 채팅방과 대화 내용이 남아 있어야 한다.
5. 현재 채팅방의 이전 대화를 OpenAI API에 함께 전달하여 대화의 문맥이 이어지도록 구현하시오.

---

### 요구사항 4. 사이드바에서 채팅방을 관리하시오.

Streamlit 왼쪽 사이드바에 다음 기능을 구현하시오.

1. 새로운 채팅방 생성
2. 저장된 채팅방 목록 조회
3. 사용할 채팅방 선택
4. 선택한 채팅방의 이전 대화 불러오기
5. 선택한 채팅방 삭제

채팅방을 삭제하면 해당 채팅방에 저장된 메시지도 함께 삭제되어야 한다.

---

### 요구사항 5. 대화 전체 비우기 기능을 구현하시오.

1. 현재 선택한 채팅방의 대화 내용을 모두 삭제할 수 있어야 한다.
2. 대화를 비워도 채팅방 자체는 유지되어야 한다.
3. 다른 채팅방의 대화 내용은 삭제되지 않아야 한다.
4. 채팅방 삭제 기능과 대화 전체 비우기 기능을 구분하여 구현한다.

---

### 요구사항 6. Streamlit 화면 상태와 DB 데이터를 연결하시오.

1. 선택한 채팅방을 기준으로 해당 방의 메시지만 조회하시오.
2. 채팅방을 변경하면 화면에 표시되는 대화 내용도 변경되어야 한다.
3. 채팅방 생성, 선택, 삭제 또는 대화 비우기 후 화면이 올바르게 갱신되어야 한다.
4. Streamlit이 사용자 입력마다 코드를 다시 실행하는 특성을 고려하여 구현하시오.

---

## 4. Agent 활용 조건

코딩 Agent에게 요구사항을 전달하여 애플리케이션 구현에 활용하시오. 단, Agent가 만든 코드를 그대로 제출하는 것에 그치지 말고 다음 작업을 수행해야 한다.

1. Agent에게 전달할 요구사항을 구체적으로 작성한다.
2. Agent가 생성한 파일과 코드를 직접 확인한다.
3. 실행 중 발생한 오류를 Agent와 함께 해결한다.
4. 데이터베이스 구조와 주요 함수의 역할을 이해한다.
5. 최종 코드가 모든 필수 요구사항을 만족하는지 직접 테스트한다.

## 5. 제출물

다음 파일을 하나의 프로젝트 폴더에 포함하여 제출하시오.

1. 실행 가능한 Streamlit 애플리케이션 코드
2. `requirements.txt`
3. `.env.example`
4. 실행 중 자동 생성되는 SQLite 데이터베이스 파일
5. `REPORT.md`

실제 API 키가 포함된 `.env` 파일은 제출하지 않는다.

## 6. REPORT.md 작성 요구사항

`REPORT.md`에는 다음 내용을 포함하시오.

### 1. Agent 활용 과정

- Agent에게 전달한 주요 요구사항
- Agent와 주고받으며 구현한 과정
- Agent가 만든 결과를 직접 검토하거나 수정한 내용

### 2. 새롭게 배운 내용

- 프로젝트를 진행하며 새롭게 배운 기술이나 개념
- Streamlit, OpenAI API, SQLite를 함께 사용하며 알게 된 점

### 3. 프로그램 동작 원리

- Streamlit이 코드를 다시 실행하는 방식과 화면 상태 관리 원리
- 채팅방과 메시지 테이블의 관계
- 메시지 저장, 조회, 대화 비우기, 채팅방 삭제 원리
- 선택한 채팅방의 이전 대화를 불러오는 원리
- 이전 대화 문맥을 OpenAI API에 전달하는 원리
- 사용자 입력부터 AI 답변 저장까지의 전체 코드 흐름

### 4. 오류 해결 및 개선 과정

- 구현 중 발생한 오류
- 오류의 원인과 해결 방법
- Agent의 제안을 그대로 사용하지 않고 직접 판단하여 수정한 내용

## 8. 실행 예시

필요한 패키지를 설치한 뒤 다음 명령으로 실행하시오.

```bash
pip install -r requirements.txt
streamlit run app.py
```

애플리케이션 파일명이 `app.py`가 아니라면 실제 파일명에 맞게 명령을 수정한다.

필수 요구사항을 모두 구현한 뒤 다음 기능에 도전할 수 있다.

1. 첫 번째 질문을 이용한 채팅방 이름 자동 생성

3. AI 답변 생성 중 로딩 상태 표시
